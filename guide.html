Innan du börjar — installera dessa program

Du behöver tre program installerade på datorn. Allt annat (Node.js-paket, databas, webbserver) sköts automatiskt av Docker.

Krävs
Docker Desktop

Kör hela applikationen (frontend, backend, databas) i isolerade containers. Det enda du verkligen måste ha.

→ docker.com/products/docker-desktop

Krävs för kodning
Git

Versionshantering och samarbete via GitHub. Används för att dela koden i gruppen.

→ git-scm.com/downloads

Rekommenderas
VS Code

Kodredigerare med utmärkt stöd för TypeScript, React och Docker. Gratis från Microsoft.

→ code.visualstudio.com

Valfritt (lokal dev)
Node.js 20+

Behövs bara om du vill köra frontend/backend direkt på datorn utan Docker. Annars sköter Docker det.

→ nodejs.org

Installationsordning: Installera Docker Desktop först → starta det → verifiera att det fungerar. Git och VS Code kan installeras i valfri ordning.

Allt annat installeras automatiskt

Prisma, Express, React, Tailwind och alla andra paket behöver du inte installera manuellt. När du kör docker compose up --build körs npm install automatiskt inuti Docker och hämtar allt från package.json.

Frontend — installeras automatiskt

Hur det fungerar: Docker läser package.json och kör npm install vid bygget. Du behöver aldrig manuellt installera Prisma, React eller något annat paket — det sker automatiskt varje gång du kör docker compose up --build.

Verifiera att allt fungerar (kör i terminalen):

docker --version        # ska visa: Docker version 24.x eller högre
docker compose version  # ska visa: Docker Compose version 2.x
git --version           # ska visa: git version 2.x

Starta projektet (efter att allt är installerat):

# 1. Hämta koden (första gången)
git clone https://github.com/GRUPPENS-REPO.git
cd "Projekt X"

# 2. Skapa miljöfil
copy .env.example .env       # Windows
# cp .env.example .env       # Mac/Linux

# 3. Bygg och starta (tar ~3–5 min första gången)
docker compose up --build

# 4. Ladda in exempeldata (kör bara en gång)
docker compose exec backend npm run seed

# 5. Öppna i webbläsaren
# http://localhost:3000
Vanligt fel: Om Docker Desktop inte är igång när du kör docker compose up får du ett felmeddelande. Starta alltid Docker Desktop först och vänta tills ikonen i aktivitetsfältet slutar snurra.

Val av teknikstack

Vi valde en modern full-stack JavaScript/TypeScript-lösning. Det betyder samma språk i frontend och backend — enklare att byta mellan dem och dela kunskap i gruppen.

Frontend
React + TypeScript

Komponentbaserat UI. TypeScript ger typsäkerhet och autocompletion vilket minskar buggar i ett stort projekt.

Build-verktyg
Vite

Snabbare än Create React App. Startar på sekunder och har inbyggd proxy mot backend under utveckling.

Styling
Tailwind CSS

Utility-first CSS. Bygger UI direkt i JSX utan att skriva separata CSS-filer. Snabbt och konsekvent.

Backend
Node.js + Express

Samma språk som frontend. Express är minimalt och flexibelt — enkelt att förstå och utöka.

Databas
PostgreSQL

Robust relationsdatabas. Branschstandard för komplexa datamodeller med relationer och beroenden.

ORM
Prisma

Skriver JavaScript istället för SQL. Definiera tabeller som kod, TypeScript-typer genereras automatiskt.

Systemarkitektur

Tre separata tjänster kommunicerar med varandra. Varje tjänst körs i sin egen Docker-container.

Frontend
React · Nginx · :3000
→
Backend
Express · Node.js · :4000
→
Databas
PostgreSQL · :5432
Hur det hänger ihop: Användaren öppnar webbläsaren på port 3000. React-appen skickar API-anrop till Express på port 4000. Express pratar med PostgreSQL via Prisma. Allt körs lokalt i Docker.

Sätta upp Docker Compose

docker-compose.yml i projektets rot definierar alla tre tjänster. Med ett kommando startar allt.

# docker-compose.yml
services:
  db:
    image: postgres:16-alpine
    environment:
      POSTGRES_USER: ${POSTGRES_USER}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
      POSTGRES_DB: ${POSTGRES_DB}
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${POSTGRES_USER} -d ${POSTGRES_DB}"]

  backend:
    build: ./backend
    depends_on:
      db:
        condition: service_healthy   # väntar på att databasen är redo
    environment:
      DATABASE_URL: postgresql://${POSTGRES_USER}:${POSTGRES_PASSWORD}@db:5432/${POSTGRES_DB}

  frontend:
    build: ./frontend
    depends_on:
      - backend
    ports:
      - "3000:80"
Viktigt: Skapa alltid en .env-fil från .env.example innan du startar. Filen ska aldrig pushas till GitHub — den innehåller lösenord. 
# .env.example — mall som delas på GitHub
POSTGRES_USER=projektx
POSTGRES_PASSWORD=projektx_secret
POSTGRES_DB=projektx_db
JWT_SECRET=byt_ut_detta_till_nagot_hemligt

Sätta upp backend (Express + Prisma)

Backend är en Express-app i TypeScript. Prisma hanterar databasen och swagger-ui-express genererar API-dokumentation automatiskt.

Mappstruktur backend:

backend/
  src/
    index.ts        ← Express startar här, Swagger konfigureras
    lib/
      prisma.ts     ← Databaskoppling (singleton)
    routes/
      projects.ts   ← GET/POST/PUT/DELETE /api/projects
      activities.ts  ← Aktiviteter + beroenden
      resources.ts   ← Resurshierarki
      raid.ts       ← Risker, issues, beslut
      share.ts      ← Read-only delning via token
  prisma/
    schema.prisma  ← Databasmodeller definieras här
    seed.ts       ← Exempeldata

Databaskoppling (Prisma singleton):

// src/lib/prisma.ts
import { PrismaClient } from '@prisma/client'

const prisma = new PrismaClient()
export default prisma

En typisk route (hämta alla projekt):

// src/routes/projects.ts
import { Router } from 'express'
import prisma from '../lib/prisma'

const router = Router()

/**
 * @swagger
 * /api/projects:
 *   get:
 *     summary: Lista alla projekt
 */
router.get('/', async (req, res) => {
  const projects = await prisma.project.findMany()
  res.json(projects)
})

export default router

Prisma-schema (exempel på modeller):

// prisma/schema.prisma
model Project {
  id          Int        @id @default(autoincrement())
  name        String
  status      String     @default("GREEN")  // GREEN, YELLOW, RED
  budget      Float?
  createdAt   DateTime   @default(now())
  updatedAt   DateTime   @updatedAt

  activities  Activity[]
  risks       Risk[]
}

model Activity {
  id          Int      @id @default(autoincrement())
  project     Project  @relation(fields: [projectId], references: [id])
  projectId   Int
  name        String
  startDate   DateTime
  endDate     DateTime
  isMilestone Boolean  @default(false)
}

Backend Dockerfile:

FROM node:20-alpine
RUN apk add --no-cache openssl   # Prisma kräver OpenSSL på Alpine
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY prisma ./prisma
RUN npx prisma generate
COPY . .
RUN npm run build
CMD sh -c "npx prisma db push && node dist/index.js"

Sätta upp frontend (React + Vite + Tailwind)

Frontend är en React SPA (Single Page Application). React Router hanterar navigering mellan sidor. React Query hanterar API-anrop och caching.

Mappstruktur frontend:

frontend/src/
  main.tsx           ← React monteras på #root
  App.tsx            ← Routing (URL → sida)
  index.css          ← Tailwind importeras
  lib/
    api.ts           ← Alla API-anrop samlade
    utils.ts         ← Färger och labels för status
  components/
    Layout.tsx       ← Sidolayout med navigation
    StatusBadge.tsx  ← Återanvändbar statustagg
  pages/
    Portfolio.tsx    ← Projektlista
    Plan.tsx        ← Gantt-vy
    Resources.tsx   ← Resurshierarki
    Management.tsx  ← Dashboard + PDF-export
    ShareView.tsx   ← Read-only extern vy

Routing i App.tsx:

// src/App.tsx
export default function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Layout />}>
          <Route index element={<Portfolio />} />
          <Route path="projects/:id" element={<ProjectOverview />} />
          <Route path="projects/:id/plan" element={<Plan />} />
        </Route>
      </Routes>
    </BrowserRouter>
  )
}

API-anrop med React Query:

// Hämta data från backend
const { data: projects } = useQuery({
  queryKey: ['projects'],
  queryFn: () => getProjects()
})

// Skicka data till backend (skapa/uppdatera)
const mutation = useMutation({
  mutationFn: (data) => createProject(data),
  onSuccess: () => qc.invalidateQueries({ queryKey: ['projects'] })

Frontend Dockerfile (multi-stage build):

# Steg 1: Bygg React-appen
FROM node:20-alpine AS build
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build   # genererar statiska filer i /dist

# Steg 2: Servera med Nginx
FROM nginx:alpine
COPY --from=build /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf

Starta applikationen

Allt startas med ett enda kommando. Docker Compose sköter ordningen — databasen startar först, sedan backend, sedan frontend.

  1. Kopiera miljövariabler: copy .env.example .env

  2. Starta och bygg alla tjänster: docker compose up --build

  3. Ladda in exempeldata (en gång): docker compose exec backend npm run seed

  4. Öppna webbläsaren på http://localhost:3000

# Starta appen
docker compose up

# Starta och bygg om (efter kodändringar)
docker compose up --build

# Stoppa appen
docker compose down

# Se loggar
docker compose logs -f backend
Tillgängliga adresser när appen körs:
  Frontend: http://localhost:3000
  API: http://localhost:4000/api
  Swagger-docs: http://localhost:4000/api/docs

Samarbeta med GitHub

Koden delas via GitHub. Varje person kör sin egen databas lokalt i Docker. GitHub innehåller aldrig databas eller lösenord.

# En person — ladda upp en gång
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/NAMN/REPO.git
git push -u origin main

# Övriga i gruppen — hämta och starta
git clone https://github.com/NAMN/REPO.git
copy .env.example .env
docker compose up --build
docker compose exec backend npm run seed
# Daglig workflow
git pull                      # hämta andras ändringar
# ...koda...
git add .
git commit -m "Beskrivning"
git push

# När någon pushat ny kod
git pull
docker compose up --build    # bygg om med ny kod
Aldrig pusha: .env-filen (lösenord), node_modules/ (för stor), dist/ (genereras vid build). Dessa står i .gitignore.
